# Level 5 Data Integrity Notes

This note addresses QA feedback that testnet and synthetic proof labels must be explicit.

## Labels Used In This Submission

- `Synthetic QA participant`: a locally generated reviewer-validation identity.
- `Stellar testnet public key`: a public address used for testnet proof rows.
- `Representative transaction`: a Stellar testnet transaction hash already referenced in app proof material.
- `Feedback response`: a synthetic QA record used to validate the feedback and iteration workflow.

## What This Is Not

- It is not proof of 50 real external users.
- It is not mainnet activity.
- It is not production financial activity.
- It does not contain private keys or mnemonics.

## Review Guidance

Use the CSV, screenshots, and transaction docs to verify that the product can collect, display, and summarize Level 5-style proof. Do not describe the synthetic QA cohort as real user acquisition.
